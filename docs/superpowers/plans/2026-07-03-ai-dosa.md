# 고블럽 도사 AI 심층 풀이 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 사주 결과에서 버튼 한 번으로 Claude(opus-4-8)가 엔진 계산 데이터 기반 개인화 풀이를 스트리밍 생성. 키는 Vercel 환경변수에만.

**Architecture:** Vercel Node Function `api/saju-reading.js`(@anthropic-ai/sdk, 스트리밍 패스스루) + 클라이언트 `play/js/dosa.js`(페이로드 구성·캐시·쿨다운·스트림 렌더) + saju.html 버튼.

**Tech Stack:** @anthropic-ai/sdk(Node, ESM), Vercel Functions, 기존 정적 사이트.

**주의:** `claude-opus-4-8`은 `temperature`/`top_p`/`budget_tokens` 미지원(400) — 사용하지 않는다. 시스템 프롬프트는 `cache_control: {type:"ephemeral"}`로 캐싱한다. 키가 없으면 501을 반환한다.

- [ ] **Step 3: `api/saju-reading.js` 작성** — 전체 코드는 실행 시 작성하는 파일과 동일해야 하며 핵심은:
  - CORS: `https://goblub.vercel.app`, `https://hiy3333.github.io`, `http://localhost:8777`, `http://localhost:3000`. OPTIONS 204.
  - POST만 허용. `process.env.ANTHROPIC_API_KEY` 없으면 501 `{error:"not_configured"}`.
  - 페이로드 검증: 간지 2글자(천간×지지 문자 집합), 기둥 3~4개, 십성 문자열 화이트리스트, 오행 숫자 5개, 신강약 라벨, 대운 리스트(1~10개). 실패 시 400.
  - `client.messages.stream({model:"claude-opus-4-8", max_tokens:2000, system:[{type:"text", text:SYSTEM, cache_control:{type:"ephemeral"}}], messages:[{role:"user", content:"다음은 만세력 엔진이 계산한 사주 데이터입니다...\n" + JSON.stringify(saju)}]})`
  - `stream.on("text", t => res.write(t))` → `finalMessage()` 후 `res.end()`. refusal이면 안내 한 줄 write.
  - 에러: 헤더 전송 전이면 502 `{error:"busy"}`.

### Task 2: 클라이언트 — dosa.js + saju.html 버튼

**Files:**
- Create: `play/js/dosa.js` — `window.Dosa = { buildPayload(r, birth, gender, trueSolar), fetchReading(payload, {onChunk,onDone,onError}) }`
  - buildPayload: 스펙의 페이로드 형태로 전부 한글 문자열 변환(Saju.GAN/JI/SIPSEONG/pillarText 사용).
  - fetchReading: localStorage 캐시(`goblub_dosa_<djb2 해시>`) 우선 → 쿨다운 60초 검사(`goblub_dosa_last`) → fetch POST → `resp.body.getReader()`로 청크 렌더 → 완료 시 캐시 저장. 501→"not_configured", 그 외 실패→"busy", 쿨다운→"cooldown"(남은 초).
- Modify: `play/saju.html`
  - 결과 패널 `r-daeun` 뒤에 dosa 박스: 버튼(🧙 고블럽 도사의 심층 풀이 듣기) + 상태문구 + 스트림 텍스트 div(`white-space:pre-wrap`).
  - btn-go 핸들러에서 `lastCtx = {r, birth, gender, trueSolar}` 저장 + dosa 박스 리셋.
  - btn-dosa 클릭 → buildPayload → fetchReading → onChunk마다 textContent 누적.
  - `<script src="js/dosa.js"></script>` 추가.
