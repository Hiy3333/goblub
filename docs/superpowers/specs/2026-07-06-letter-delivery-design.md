## 목적

만든 사람이 유형·상대이름·메시지를 정해 링크를 만들어 보내면, 받은 사람이 플레이(YES 유도 개그 포함)하고 날짜·시간을 잡은 뒤, 결과 이미지 저장 또는 되답 링크로 만든 사람에게 답을 전달한다. 백엔드·API 키 없음(URL 인코딩 + Canvas).

## 단일 파일 3상태 (`play/letter.html`)

URL 해시 파라미터로 상태 분기:
- 파라미터 없음 → **만들기 모드**
- `#d=<base64url(JSON)>` → **플레이 모드** (받은 사람)
- `#d=...&r=<base64url(JSON)>` → **결과 모드** (플레이 완료 화면을 바로 렌더; 만든 사람이 되답 링크로 접속 시)

### 인코딩
- `encode(obj)` = `btoa(encodeURIComponent(JSON.stringify(obj)))`를 URL-safe 치환(`+`→`-`, `/`→`_`, `=` 제거), `decode`는 역변환. 손상 시 만들기 모드로 폴백.
- `d` = `{ t:"date"|"apology", n:"상대이름"(≤20), m:"한줄메시지"(≤60, 선택) }`
- `r` = `{ menu:"🍴 한식"(date만), day:"YYYY-MM-DD", time:"HH:MM" }`

## 유형
| 유형 | 질문 기본값 | 단계 |
|------|------------|------|
| date(데이트 신청 💚) | "나랑 데이트 갈래?" | 메뉴 → 날짜 → 시간 |
| apology(사과하기 🍎) | "내 사과 받아줄래?" | 날짜 → 시간 (화해 약속) |
- 메시지(m)가 있으면 질문 위에 한 줄로 표시.
- 메뉴 6종: 🍚한식 🥢중식 🍣일식 🍝양식 🍜분식 ☕카페

- 유효성: 이름 필수(공백 불가). 초과 길이 자름.

## 플레이 모드 UI (받은 사람)
- 타이핑 인트로: 검은 터미널 폐기. goblub 톤 — 카드 안에서 "👾 고블럽이 편지를 배달하는 중…" + 점 애니메이션 3~4줄 순차 표시 후 질문 등장.
- 질문 + [YES(응! 좋아 💚 / 사과 받을게 🍎)] + [싫어/안 받아] 버튼.
  - **도망 버튼 유지**: 원전의 glideFrom 로직 이식(마우스 근접·터치 시 도망, 화면 경계 회피). goblub 톤 .btn-ghost 스타일.
- YES 클릭 → 단계 진행. 메뉴(date)=타일 그리드. 날짜=동적 달력. 시간=슬롯 그리드(11·12·13·14·17·18·19·20시).
