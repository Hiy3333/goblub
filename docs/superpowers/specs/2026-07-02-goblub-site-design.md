# goblub — 즐길거리 포털 사이트 설계

날짜: 2026-07-02
상태: 승인됨

## 목적

사람들이 와서 가볍게 즐길 수 있는 콘텐츠 포털. 사주, 심리테스트 같은 인터랙티브 HTML 페이지를 하나씩 만들어 붙이고, 직접 만든 모바일 앱의 다운로드 링크도 제공한다.

## 범위 (첫 버전)

- 메인 페이지: 로고 + 콘텐츠 카드 그리드
- 앱 다운로드 페이지: 앱 카드 틀 (내용·링크는 추후 제공받아 채움)
- 사주·심리테스트: 이번 버전에서는 "준비 중" 카드만. 실제 콘텐츠는 다음 사이클
- 배포: GitHub Pages (전용 저장소 `goblub`, 푸시 전 사용자 확인)

## 결정 사항

| 항목 | 결정 |
|------|------|
| 사이트 이름 | goblub |
| 기술 | 순수 정적 HTML/CSS/JS, 빌드 도구 없음 |
| 디자인 톤 | 밝고 알록달록 (놀이공원 느낌, 둥근 카드, 이모지 활용) |
| 배포 | GitHub Pages, `https://<계정>.github.io/goblub/` |
| APK 제공 | 다운로드 링크 (사용자가 추후 제공) |

## 구조

```
html/  (= goblub 저장소 루트)
├── index.html          메인: 로고 + 콘텐츠 카드 그리드
├── apps.html           앱 다운로드: 앱 카드(아이콘·이름·소개·다운로드 버튼)
├── css/style.css       공용 스타일 (색상 팔레트, 카드, 반응형)
├── js/common.js        공용 헤더/푸터 삽입
└── play/               즐길거리 페이지 폴더 (사주·심리테스트 추가 예정)
```

## 페이지 상세

### index.html
- 상단: goblub 로고(텍스트 로고) + 캐치프레이즈
- 본문: 콘텐츠 카드 그리드
  - 📱 앱 다운로드 → `apps.html` (활성)
  - 🔮 사주 → 준비 중 배지, 클릭 불가
  - 🧠 심리테스트 → 준비 중 배지, 클릭 불가
- 카드 하나 추가 = 새 콘텐츠 연동, 이 규칙만 지키면 확장 끝

### apps.html
- 앱 카드 2~3개 틀: 아이콘 자리, 앱 이름, 한 줄 소개, 다운로드 버튼(준비 중 상태)
- 링크를 받으면 버튼 href만 채우면 되는 구조

### 공용
- `css/style.css`: CSS 변수로 팔레트 정의, 카드 컴포넌트, 모바일 반응형(그리드 1~3열)
- `js/common.js`: 헤더(로고+홈 링크)·푸터를 각 페이지에 삽입해 중복 최소화

## 에러 처리
- 정적 사이트라 서버 에러 없음. 준비 중 카드는 시각적으로 비활성 처리(커서·투명도)로 잘못된 클릭 방지.

## 테스트/검증
- 로컬 브라우저 미리보기로 데스크톱(1280px)·모바일(375px) 확인
- 링크 동작(메인 ↔ 앱 페이지) 확인
- 배포 후 GitHub Pages URL 접속 확인
