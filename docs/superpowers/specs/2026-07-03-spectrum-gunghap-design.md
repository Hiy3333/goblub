## A. 🌈 스펙트럼 찾기 (`play/spectrum.html`)

### 원칙
- 근거: 킨제이 스케일·클라인 그리드의 자기보고 프레임. 고정관념 문항 금지 — 오직 자기 경험·느낌.
- 정체성 자기결정 원칙을 인트로에 명시. 결과는 진단이 아닌 "오늘의 나를 비추는 거울".
- 프라이버시: 응답·결과 서버 전송 없음, localStorage 저장도 없음, 결과 링크 공유 기능 의도적 제외(아웃팅 경로 차단). 인트로에 "이 브라우저 화면에만 표시되고 어디에도 저장되지 않아요" 명시.

### 구조
- 12문항 = 3축 × 4문항. 진행 UI는 기존 퀴즈 스타일(진행바 + 선택 카드).
  - 끌림 축(4): 설렘·눈길·상상 속 대상의 젠더 방향
  - 로맨스 축(4): 연애 감정·설렘의 대상
  - 자기인식 축(4): 라벨 필요성, 변화 가능성, 탐색 개방성
- 선택지 스코어 키: `same`(같은 젠더), `other`(다른 젠더), `both`(둘 다/무관), `ace`(거의 안 느낌), `fluid`(모르겠음/변함)
  - same/other 단독 +2, both는 same+1 other+1, ace +2, fluid +1(자기인식 축은 fluid/고정 스코어 중심)

### 결과 계산
- `pos = same / (same + other) * 100` (분모 0이면 에이스 중심 결과)
- 프로필: pos<20 "한끝 자리(다른 젠더 쪽)", 20~40 "기울어진 들판(다른 젠더 쪽)", 40~60 "가운데 들판", 60~80 "기울어진 들판(같은 젠더 쪽)", >80 "한끝 자리(같은 젠더 쪽)"
- ace 점수 ≥ 6 → "고요한 호수" 프로필로 대체(에이스 스펙트럼 소개). ace 3~5 → 보조 코멘트.
- fluid ≥ 5 → "아직 그리는 중" 보조 코멘트(유동성·탐색 존중).
- 표시: 무지개 그라데이션 바(CSS linear-gradient) 위 위치 마커 + 프로필 이모지·이름·묘사 + 관련 용어 소개("이런 단어들이 네 곁에 있어: 양성애, 범성애…" — 프로필별 매핑) + 마지막 고정 문구 "라벨은 도구일 뿐, 주인은 언제나 너야."
- 하단 한 줄: "더 알아보고 싶다면 — 한국성적소수자문화인권센터, 청소년성소수자위기지원센터 띵동" (텍스트 링크)
- 버튼: 다시하기만. 공유·저장 없음.

### 검증
- 전부 '다른 젠더' 응답 → pos<20 프로필, 전부 '같은 젠더' → pos>80, 전부 both → 가운데, 전부 ace → 고요한 호수
- 저장 부재 확인(localStorage에 어떤 키도 안 씀), 모바일 375px

## B. 💑 우리 궁합 (`play/gunghap.html` + `play/js/gunghap.js` + `api/gunghap-reading.js`)

### 입력
- 나: 년/월/일/시(몰라요 가능)/성별(선택). `goblub_birth` 있으면 프리필. 상대: 동일 폼. 상대 정보는 저장하지 않음.

### 엔진 점수 (gunghap.js, 100점 만점)
기존 `Saju.compute` 결과 두 개(rA, rB)로 계산:
1. **일간 케미 30점**: 천간합 쌍(갑기, 을경, 병신, 정임, 무계 — 인덱스 (a,b): a%5 쌍 && 음양 다름 == (b-a+10)%10===5) → 30 / 상생(어느 한쪽이 다른 쪽을 生: 오행 rel 1 또는 4) → 24 / 같은 오행(비겁) → 18 / 상극(rel 2,3) → 13
2. **일지 케미 30점**: 육합 → 30 / 삼합 그룹 동일(신자진·해묘미·인오술·사유축) → 26 / 충 → 8 / 그 외 → 18
3. **오행 보완 25점**: A에서 0인 오행을 B가 1.0 이상 보유(및 반대)한 개수 n → 25점 만점 매핑(`13 + 4*n`, 최대 25, 결핍이 없으면 균형 보너스 20). 구현식은 계획서에 확정.
4. **띠 케미 15점**: 년지 육합 15 / 삼합 13 / 충 4 / 그 외 10
- 등급: 90+ 천생연분 / 75+ 찰떡궁합 / 60+ 노력하면 꿀떡 / 45+ 서로 배우는 사이 / 그 외 파란만장 드라마
- 풀이: 항목별 결과 유형(합/생/비겁/극, 육합/삼합/충/평 등)마다 고정 텍스트 1~2문장 + 등급 총평 + 조언 1줄

### UI
- 결과: 점수 대형 + 게이지 바 + 등급명 + 두 사람 일주(한자) 나란히 + 항목별 풀이 4줄 + 조언
- **🧧 부적 카드**(Canvas 720×1080 PNG): 노란 바탕(#ffd93d 계열)+빨간(#d64545) 테두리 문양, 상단 세로느낌 "戀愛運上昇符", 두 사람 일주 간지(한자), 하트+👾, 점수·등급, "휴대폰 배경화면으로 하면 연애운이 커진대요", goblub.vercel.app
- **🧙 도사의 궁합 리포트**: 버튼 → `api/gunghap-reading.js` 스트리밍(사주 페이지 패턴 동일: 캐시·쿨다운 60초·501 수련중·502 손님많음)

### api/gunghap-reading.js
- 페이로드: `{ gunghap: { a: <사주 페이로드(도사와 동일 형식)>, b: <동일>, score: {total, grade, ilgan, ilji, oheng, tti} } }`
- 검증: 두 사주 각각 도사 API와 동일 규칙 + score 숫자 범위.
- 시스템 프롬프트: 고블럽 도사 페르소나 유지. 사실 규칙 동일(계산 금지·데이터 근거 명시). 출력 5섹션: 💞 첫인상 케미 / 🔥 불붙는 지점 / 🌧 부딪히는 지점 / 🤝 화해 공식 / 🧙 도사의 연애 처방. 분량 600~900자. 단정 금지, 재미 톤, 마지막 "궁합은 재미로, 사랑은 두 사람의 몫으로. — 고블럽 도사".
- 모델·스트리밍·CORS·키 처리: saju-reading과 동일(`claude-opus-4-8`, max_tokens 1800).

### 메인 카드
- 💑 우리 궁합(tint-coral): "두 사람의 사주로 보는 케미 + 연애운 부적까지"
- 🌈 스펙트럼 찾기(tint-purple): "나를 비추는 거울 — 끌림의 스펙트럼 자기탐색"
- 빈 슬롯은 유지(맨 뒤).

### 검증
- 궁합: 천간합 사례(갑일간+기일간 → 일간 30점), 충 사례(자일지+오일지 → 일지 8점) 수동 대조. 같은 입력 재계산 동일. 부적 PNG 생성. AI 버튼 501 경로.
- 배포 후 두 페이지 200.
