# 신규 4종 설계 — 결정의 신 · 전생 대환장 파티 · 밸런스 게임 · 고블럽 키우기

날짜: 2026-07-04 / 상태: 승인됨(대화로 합의)

## 1. ⚡ 결정의 신 (`play/decide.html` + `api/decide.js`)

- 입력: 선택지 A, 선택지 B(각 1~200자) + 선택 "상황 한 줄"(0~200자)
- AI(Claude opus-4-8, 스트리밍): "결정의 신" 페르소나 — **반드시 하나를 단호하게 고름**. 출력 3섹션: ⚡ 판결 / 🧾 신의 논리(드립) / ⚠️ 단 하나의 조건. 300~500자. max_tokens 800
- 안전 규칙: 의료·법률·투자·위험 행동 결정 요청이면 판결 대신 "그건 신도 대신 못 정한다" + 전문가/신중 안내
- 클라이언트: dosa.js의 공용 스트림(fetchAI) 재사용 — 캐시(동일 질문)·쿨다운 60초·501 수련중 동일

## 2. 🔮 전생 대환장 파티 (`play/pastlife.html` + `api/pastlife.js`)

- 입력: 생년월일(+시 선택). 사주 페이로드(도사와 동일 빌더, 대운 제외 가능)를 시드로 전송
- AI 응답은 **마커 구분 텍스트** (클라이언트가 버퍼링 후 파싱, 스트림 중에는 "N번째 전생 발굴 중…" 진행 연출):
  - `@@LIFE|시대|이모지|칭호` 다음 줄부터 본문 2~3문장 (4~5개, 시대는 전부 다르게: 선사~미래 자유)
  - 나쁜 전생(최소 1개, 최대 2개)은 본문 뒤 `@@FIX` 다음 줄에 "고블럽이 씹어먹고 바꿔준 개선판" 본문 동봉 (재호출 없이 소환 가능)
  - `@@FINALE` 다음 줄부터: 전생들이 현생의 나로 이어지는 회수 문단 (사주 오행·일간 근거 가볍게 인용)
  - `@@SVG` 다음: 대표 전생 1장면 `<svg viewBox="0 0 600 400">…</svg>` (flat color, 외부 참조·script 금지)
- 클라이언트 렌더: 전생 카드 릴레이(시대 배지+이모지+칭호+본문). 나쁜 전생 카드에 **[👾 고블럽 소환]** → 씹어먹기 모션(글자 끝에서부터 순차 삭제 + 고블럽 이동 + 와그작) → FIX 텍스트 타이핑 인. SVG는 sanitize(script/on* 제거) 후 표시 + PNG 저장 버튼(svg→canvas)
- max_tokens 4000, 캐시(생일별)·쿨다운 공유

## 3. ⚖️ 밸런스 게임 (`play/balance.html` + `api/balance-vote.js`)

- 12라운드 고정 문항(페이지 내장, 구현 시 작성). 선택 → `POST /api/balance-vote {id, pick}` → Redis `HINCRBY vote:<id> <pick>` 후 합계 반환 → "전국 63% vs 37%" + 내가 다수/소수파 표시
- Upstash Redis REST 사용(의존성 없음, fetch로 REST 호출). 환경변수 `KV_REST_API_URL/KV_REST_API_TOKEN` 또는 `UPSTASH_REDIS_REST_URL/TOKEN` 둘 다 지원
- **미연결 시 우아한 저하**: 501 → 클라이언트는 "전국 통계 준비 중" 배지와 함께 통계 없이 진행
- 중복 방지: 문항당 1회(localStorage). 종료 화면: 소수파 비율 요약("당신은 12판 중 7판 소수파 — 청개구리 유형") + 결과 문구 복사
- CORS 동일 화이트리스트

## 4. 👾 고블럽 키우기 — 활동 먹이 모델 (`play/goblub.html` + `play/js/feed.js`)

- **먹이 획득**: 콘텐츠 결과를 볼 때마다 `GoblubFeed.grant(src)` — 소스당 하루 1개 (localStorage `goblub_pet.feeds["src_YYYYMMDD"]`)
  - 소스(10): saju, fortune, today, gunghap, mbti, love, stress, decide, pastlife, balance
  - **spectrum은 제외** (프라이버시 원칙 — 이용 흔적을 저장하지 않는 페이지)
- **성장**: 먹이 1개=경험치 1. 레벨: 알(0)→아기(3)→청소년(8)→어른(15)→대왕(25)→전설(40). 레벨별 모습은 이모지 조합+크기(🥚→👾→👾✨→👾🎩→👾👑→🌟👾🌟)
- 고블럽 페이지: 펫 표시(이름 짓기 가능) + 보유 먹이 주기(냠냠 모션+경험치 바) + 오늘 먹이 얻을 수 있는 콘텐츠 목록(미획득만, 링크) + 레벨업 연출
- 데이터는 전부 localStorage(`goblub_pet`) — 서버 없음
- 기존 페이지 연동: 각 결과 렌더 지점에 feed.js 로드+grant 1줄 (7개 페이지)
