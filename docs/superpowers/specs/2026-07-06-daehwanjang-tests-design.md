# 대환장 테스트 허브 개편 설계

**작성일:** 2026-07-06
**목표:** 노말했던 테스트 3종(연애 스타일·스트레스 유형·스펙트럼 찾기)을 대환장 MBTI처럼 재미난 컨셉으로 재기획하되, 각 테스트 성격에 맞는 고유 재미 요소를 넣고 짤 이미지 공유만 공통화한다.

## 결정 사항 요약

| 기존 | 새 컨셉 | 결과 연출 |
|---|---|---|
| 연애 스타일 | **내 안의 연애세포 회의** 💘 | 우세 세포 = 연애 타입 + 세포 비율 막대 |
| 스트레스 유형 | **스트레스 몬스터 변신** 🔥→👾 | 몬스터 공개 + 고블럽이 먹어주는 모션 |
| 스펙트럼 찾기 | **나의 무지개 좌표** 🌈 | 가로 스펙트럼 바 위 '나의 좌표' (감성·존중) |

- **AI 불필요:** 전부 클라이언트 채점(순수 JS). Gemini/API 호출 없음 → 즉시·무료·키 무관.
- **공통:** 짤 이미지(Canvas PNG) 공유는 `sharecard.js` 하나로 통일.
- **미사용:** 결과별 해시 공유 링크는 이번 범위 밖(YAGNI). 테스트 페이지 URL 복사만 유지.

## 아키텍처 — 하이브리드

세 테스트는 **흐름(인트로 → 상황극 질문 → 채점)**은 동일하지만 **결과 렌더링**이 완전히 다르다. 따라서 공유할 것만 공유한다.

### 1) `play/js/quiz.js` — 흐름+채점 엔진으로 업그레이드
기존 quiz.js는 `resolve→results` 맵(이산 결과)만 지원한다. 이를 일반화한다.

```
Quiz.init(el, config)
config = {
  emoji, title, intro,                         // 시작 화면
  feedSrc,                                      // (선택) 있으면 결과 도달 시 GoblubFeed.grant(feedSrc)
  questions: [{ q, choices: [{ text, scores: {키:점수} }] }],
  renderResult: function(totals, el, api)       // ★ 테스트별 커스텀 결과 렌더러
}
api = { restart: fn, shareLink: fn(btn) }       // 결과 렌더러가 쓰는 공통 액션
```

- 엔진 책임: 시작 화면, 질문 스테퍼, 진행 바, 점수 누적(`totals`), 마지막에 `config.renderResult(totals, el, api)` 호출.
- `feedSrc`가 있으면 결과 진입 시 1회 `GoblubFeed.grant(feedSrc)` (love/stress만 사용, spectrum은 미지정).
- 하위호환: 기존 `resolve`/`results` 방식은 제거하고 세 테스트를 모두 `renderResult`로 재작성한다(현재 quiz.js를 쓰는 건 love/stress뿐이며 이번에 전부 교체).

### 2) `play/js/sharecard.js` — 공용 짤 이미지 모듈 (신규)
```
ShareCard.render(spec) -> <div> (미리보기 + [이미지 저장] 버튼 포함)
spec = {
  palette: "coral"|"yellow"|"purple"|...,   // 배경 톤(style.css 팔레트)
  emoji,                                     // 큰 대표 이모지(또는 고블럽 아트 대신 이모지)
  badge,                                     // 상단 소제목 (예: "연애세포 회의 결과")
  title,                                     // 결과 이름 (예: "밀당세포 우세형")
  lines: [문자열...],                          // 본문 2~3줄
  footer: "goblub · 대환장 테스트"
}
```
- Canvas 2D → `toBlob` PNG 다운로드(기존 letter/mbti 카드 패턴 재사용, iOS `toDataURL` 폴백 포함).
- 세로 카드(예: 640×800). Jua 폰트, 고블럽 팔레트. 이모지·한글 렌더 확인.

### 3) 각 테스트 페이지 = 얇은 config + 커스텀 renderResult
`love-test.html` / `stress-test.html` / `spectrum.html` 각각이 자기 questions와 renderResult만 정의하고 quiz.js·sharecard.js를 로드한다.

## 테스트별 상세

### ① 내 안의 연애세포 회의 (love-test.html)
- **연애세포 6종(결과 타입):**
  1. 밀당세포 — 밀고 당기기의 고수, 여유로운 척 실은 계산 중
  2. 직진세포 — 좋으면 바로 돌진, 고백도 선빵
  3. 집착세포 — 사랑하면 올인, 읽씹·카톡 1을 못 견딤
  4. 모솔세포 — 설렘 상상력은 100, 실행력은 0
  5. 츤데레세포 — 좋아할수록 퉁명, 표현이 반대로
  6. 헌신세포 — 다 퍼주는 사랑, 내 것보다 상대 먼저
- **질문:** 상황극 10문항. 각 선택지가 세포들에 점수 배분. 회의체 톤(예: *"썸남 답장 도착. 회의실이 술렁인다. 내 손가락은?"*).
- **결과(renderResult):** 최고점 세포 = 나의 연애 타입.
  - 팩폭 설명 문단(2~3문장) + **세포 비율 막대**(상위 3~4세포 %) + "환상의 짝꿍세포 / 상극세포" 한 줄.
  - 짤 카드: badge "연애세포 회의 결과", title "○○세포 우세형", lines 팩폭 요약 + 짝꿍/상극.
  - `feedSrc: "love"`.

### ② 스트레스 몬스터 변신 (stress-test.html)
- **스트레스 몬스터 6종(결과 타입):**
  1. 폭발몬 🌋 — 버럭, 참지 않고 터뜨림
  2. 잠수몬 🌊 — 잠수·회피, 연락 두절
  3. 눈물몬 💧 — 펑펑 울음으로 배출
  4. 먹보몬 🍔 — 폭식·보상소비
  5. 예민몬 🌵 — 가시 바짝, 짜증·날카로움
  6. 얼음몬 🧊 — 셧다운·무기력·번아웃
- **질문:** 상황극 10문항. 스트레스 상황 선택지가 몬스터에 점수.
- **결과(renderResult):** 최고점 몬스터 공개 + 팩폭 설명.
  - **"고블럽 소환" 버튼 → 고블럽 아트가 등장해 그 몬스터 이모지를 냠 먹는 모션**(pastlife 씹어먹기 + goblub-art 재사용) → 진정 처방 한 줄로 전환.
  - 짤 카드: badge "나의 스트레스 몬스터", title "○○몬", lines 팩폭 + 처방.
  - `feedSrc: "stress"`. (고블럽 세계관 직결 — 먹이 연동 유지)

### ③ 나의 무지개 좌표 (spectrum.html)
- **연출:** 부드러운 상황극 10~12문항. 톤은 감성·힐링(개그 최소).
- **채점/결과:** 이산 타입이 아니라 **연속 좌표**. 선택지 점수를 합산해 0~100 스펙트럼 위치를 만든다.
  - 결과 = **가로 스펙트럼 바 + '나의 좌표' 마커** + 다정한 한 줄(대략적 구간에 맞춘 긍정 메시지, 임상 용어·등급·우열 없음).
  - 예: *"무지개의 어디쯤에 있어도 괜찮아요. 오늘 당신의 좌표는 여기예요."*
- **프라이버시(기존 원칙 유지·강화):**
  - 결과 **저장 안 함**(localStorage 미기록), **goblub 먹이(feed) 제외**(`feedSrc` 미지정).
  - 짤 공유는 **opt-in**(자동 생성 X, 버튼을 눌러야 카드 생성). 카드 문구는 특정 지향 표현 없이 은은하게(아웃팅 방지) — 좌표 수치/라벨을 카드에 크게 박지 않고 "나의 무지개 좌표" 정도로.
  - 하단 안내 문구: 재미·자기탐색용이며 진단 아님.

## 공통 — 테스트 허브 & 연동
- **tests.html:** 3칸 타일의 제목·이모지·설명을 새 컨셉으로 교체(대환장 MBTI 타일은 유지).
  - 연애세포 회의 / 스트레스 몬스터 변신 / 나의 무지개 좌표
- **feed.js SOURCES:** love·stress 라벨을 새 이름으로 갱신. spectrum은 계속 제외.
- **메인 index.html:** "대환장 테스트" 허브 카드 문구는 그대로(내부만 바뀜) — 필요 시 카피 소폭 조정.

## 데이터/상태
- love·stress: 결과는 세션 내에서만(새로고침 시 리셋). 채점은 순수 함수.
- spectrum: 상태 저장 없음.
- 짤 이미지: 클라이언트 Canvas 생성, 서버 전송 없음.

## 에러/엣지
- 짤 저장: `toBlob` 미지원 브라우저 → `toDataURL` 폴백. 실패 시 안내 문구.
- 동점 처리: love/stress 최고점 동점 시 정의된 우선순위(배열 순서 앞선 타입) 채택.
- 질문 도중 이탈/재시작: `renderResult`가 받는 `api.restart`로 처음부터.

## 범위 밖(하지 않음)
- 결과별 해시 공유 링크(대환장 MBTI엔 있으나 이번 3종엔 미포함).
- 전국 통계(Upstash) 연동.
- AI 해설.
- 생존 스탯·단톡방 반응(각 테스트 고유 요소로 대체하므로 일괄 적용 안 함).
