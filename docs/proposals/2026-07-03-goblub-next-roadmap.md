# goblub 차기 기능 기획서

날짜: 2026-07-03
현재 상태: 사주 2.0(십성·지장간·신강약·진태양시·대운), 오늘의 운세 4종, 오늘의 사주, 심리테스트 3종, 앱 다운로드 페이지 배포 완료 (https://hiy3333.github.io/goblub/)

## 1부. 사용자 관점에서 지금 더 필요한 것

> 콘텐츠는 이미 좋다. 지금 부족한 건 **"들어오게 하고, 퍼지게 하고, 다시 오게 하는"** 장치다.

### 1-1. 퍼지게 하기 (바이럴 장치) — 최우선

| 항목 | 문제 | 해결 |
|------|------|------|
| 결과 공유 | 지금은 "링크 복사"뿐. 친구에게 내 결과를 보여줄 방법이 없다 | 결과를 **이미지 카드로 저장**(Canvas로 그려서 다운로드/길게 눌러 저장). MBTI·사주 결과가 인스타 스토리에 올라가는 순간이 최고의 광고 |
| 카톡 미리보기 | 링크를 카톡에 붙이면 밋밋한 텍스트만 보인다 | **오픈그래프(OG) 메타태그** + 대표 이미지. 페이지별 og:title/og:image 설정 (정적 HTML에 태그만 추가하면 됨, 비용 0) |
| 결과 링크 | 내 운세 링크를 보내도 상대방 화면엔 입력 폼이 뜬다 | 생일을 쿼리스트링에 인코딩(`fortune.html?b=19950315`) → 링크 열면 바로 그 사람 결과 |
| 파비콘 | 브라우저 탭이 기본 아이콘 | goblub 캐릭터 파비콘 + 애플 터치 아이콘 |

### 1-2. 다시 오게 하기 (재방문 장치)

- **홈 화면에 "오늘의 한 줄" 위젯**: 메인 상단에 오늘의 일진 + 띠별 한 줄 운세를 바로 노출. "매일 바뀌는 첫 화면"이 재방문 이유가 된다.
- **PWA(홈 화면에 추가)**: manifest.json + 아이콘만 있으면 폰 홈 화면에 앱처럼 설치 가능. 오늘의 운세와 궁합이 최고로 좋아지는 조합.
- **생일 저장 공유**: 지금은 운세 페이지만 생일을 기억한다. 사주 페이지도 같은 localStorage를 쓰게 통일 → 어느 페이지든 한 번 입력하면 끝.

### 1-3. 지켜보기 (운영 장치)

- **방문 통계**: GoatCounter(무료·스크립트 한 줄·개인정보 수집 없음) 삽입 → 어떤 콘텐츠가 인기인지 보고 다음 콘텐츠 결정.
- **앱 페이지 완성**: 다운로드 버튼이 전부 "준비 중"이면 신뢰가 떨어진다. 준비된 APK부터 링크 연결 + 앱별 스크린샷 1장.

## 2부. 새로운 재미난 기능 (API 없이, 지금 엔진으로 가능)

이미 만들어둔 사주 엔진(Saju)과 퀴즈 엔진(Quiz)을 재활용하면 아래는 전부 "하루 콘텐츠"다.

### A. 사주 엔진 재활용 (Saju/Fortune 확장)

1. **💑 궁합 보기** — 두 사람 생일 입력 → 일간 십성 관계(그가 나의 정재/정관인지!) + 띠 합충 + 오행 상호보완 점수. 커플·썸 단계에서 공유 폭발력 최고. ★강력 추천 1순위
2. **🎰 오늘의 행운** — 일진 기반 행운 숫자(로또 픽), 행운 색, 행운 방향, 행운 시간대. 오늘의 사주 페이지에 섹션 추가만 해도 됨.
3. **🗓️ 올해의 운세(세운)** — 올해 간지 vs 내 일간. 연초·생일 시즌 필수 콘텐츠.
4. **🐭 띠별 오늘의 운세** — 12띠 × 오늘 일진 관계로 한 줄씩. 포털 운세의 국룰. 메인 위젯과 연동.

### B. 퀴즈 엔진 재활용 (설정 객체만 새로 쓰면 끝)

5. **🔮 전생 테스트** — "나의 전생은?" (조선 암행어사, 이집트 고양이…) 가볍고 공유 잘 됨.
6. **🐾 나와 어울리는 동물 테스트** — goblub 몬스터 세계관과 연결 가능.
7. **⚖️ 밸런스 게임** — "평생 라면 금지 vs 평생 치킨 금지". 결과 화면에 "나는 A 선택!" 공유. (투표 집계는 3부의 서버리스가 생기면 전체 통계도 가능)

### C. 새 미니게임 (별도 페이지)

8. **⚡ 반응속도 테스트** — 초록불 뜨면 클릭. 기록을 이미지로 공유.
9. **🧠 순간 기억력 게임** — 숫자 순서 외우기. 단계별 난이도.

## 3부. 사주 × AI API 연동 아이디어 🤖

### 먼저 알아야 할 것: 정적 사이트와 API 키

goblub은 GitHub Pages(정적 호스팅)라서 **API 키를 사이트 코드에 넣으면 누구나 훔쳐갈 수 있다.** 그래서 연동 방식이 3단계로 나뉜다:

| 방식 | 원리 | 키 안전 | 비용 통제 | 난이도 |
|------|------|--------|----------|--------|
| **① 배치 사전 생성** | 내 PC에서 매일 스크립트가 AI로 콘텐츠를 만들어 JSON으로 커밋 → 사이트는 그 파일만 읽음 | 완벽(키가 사이트에 없음) | 완벽(하루 1회 호출) | 하 |
| **② 서버리스 프록시** | Cloudflare Workers(무료 10만 요청/일)에 키를 숨기고, 사이트는 프록시에 요청 | 안전 | 요청 제한 설정 필요 | 중 |
| **③ BYOK** | 사용자가 자기 API 키를 입력(localStorage) | 사용자 책임 | 사용자 부담 | 하 |

**추천 로드맵: ① 로 시작 → 반응 좋으면 ② 로 실시간화.** (③은 개발자 친구용 이스터에그 정도)

### 아이디어 목록

#### 🥇 1. "고블럽 도사" — AI 사주 심층 풀이 (추천 1호)
- 사주 결과 화면에 **"고블럽 도사에게 심층 풀이 듣기"** 버튼.
- 이미 계산된 구조화 데이터(팔자 8글자, 십성 분포, 신강약, 대운, 오행)를 프롬프트에 넣으면 AI가 **그 사람만의 장문 풀이**를 써준다. 지금의 고정 텍스트 풀이와 차원이 다른 개인화.
- 핵심: 계산은 우리 엔진(정확), 해석만 AI(풍부). AI가 사주를 계산하면 틀리지만, 우리가 계산한 걸 해석만 시키면 강력하다.
- 방식 ②(실시간). 캐릭터 설정: "나쁜 감정을 먹어주는 고블럽 도사" — 앱 세계관과 연결.

#### 🥈 2. AI 오늘의 운세 문장 — 매일 새로운 글 (추천 2호, 가장 쉬움)
- 지금 운세 텍스트는 십성별 10종 고정 → 매일 보면 반복이 보인다.
- **방식 ①**: 매일 아침 스크립트가 "오늘의 일진 × 10개 일간 × 4개 카테고리"(40개 문장)를 AI로 생성해 `fortune-daily.json`으로 커밋. 사이트는 그 파일을 읽기만.
- 키 노출 0, 비용은 하루 1회 호출(Haiku면 몇 원 수준). 스케줄러(작업 스케줄러/cron)로 완전 자동화 가능.

#### 🥉 3. 내 사주 수호 고블럽 만들기 (세계관 연동 킬러 콘텐츠)
- 사주 오행·십성으로 **나만의 고블럽 몬스터**를 정해주고, AI가 이름·성격·내가 조심할 감정·먹여줘야 할 감정을 지어준다.
- 예: "목 기운 과다 + 상관 발달 → 당신의 고블럽은 '새싹뿔 우두두'. 조급함을 먹여주세요."
- 이미지 생성 API까지 붙이면 프로필 사진급 공유 콘텐츠. (1차는 텍스트+미리 그린 오행별 캐릭터 5종으로 시작 가능 = 방식 ① 조차 불필요)
- **Goblub 앱의 세계관 예고편** 역할 — 사이트가 앱 마케팅이 된다.

#### 4. AI 궁합 리포트
- 2부의 궁합 기능(계산)에 AI 해석을 얹는다: 두 사주의 십성 관계를 주고 "이 커플의 케미, 싸울 때 패턴, 화해 공식"을 재밌게 쓰게 한다.

#### 5. 꿈해몽 도사
- 꿈 내용을 입력하면 AI가 해몽 + **내 사주 오행에 비춘 조언**("수 기운이 부족한 당신이 물꿈을 꿨다면…"). 방식 ② 필요.

#### 6. AI 작명소
- 부족한 오행을 보완하는 이름/닉네임/아이디 추천. 가볍고 재밌는 단발 콘텐츠.

### 비용·안전 가드레일 (② 실시간 방식 공통)

- 모델: Claude Haiku(빠르고 저렴) → 심층 풀이만 Sonnet.
- Workers에서 **일일 총량 제한**(예: 하루 500회) + 사용자당 쿨다운(같은 브라우저 1분 1회).
- 같은 사주 입력은 결과를 캐시(같은 팔자 = 같은 리포트 재사용).
- 프롬프트에 "의료·법률·투자 조언 금지, 재미 톤 유지" 고정.

## 4부. 추천 실행 순서

| 순서 | 내용 | 효과 | 규모 |
|------|------|------|------|
| 1 | OG 태그 + 파비콘 + 결과 이미지 저장 | 공유 시작 = 유입 시작 | 반나절 |
| 2 | 궁합 보기 + 오늘의 행운 | 체류·재방문 | 1일 |
| 3 | 메인 "오늘의 한 줄" 위젯 + PWA | 재방문 습관화 | 반나절 |
| 4 | AI 배치: 매일 새 운세 문장(방식 ①) | 콘텐츠 신선도 | 1일 |
| 5 | 수호 고블럽(텍스트판) | 세계관·앱 연계 | 1일 |
| 6 | Cloudflare Workers 프록시 + 고블럽 도사 심층 풀이(방식 ②) | 킬러 기능 | 2~3일 |
| 7 | 방문 통계 보고 다음 결정 | 데이터 기반 운영 | 즉시 |

> 원칙: **계산은 엔진, 해석은 AI, 키는 절대 사이트에 넣지 않는다.**
